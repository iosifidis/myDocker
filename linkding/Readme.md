# Linkding: Self-hosted Διαχειριστής Σελιδοδεικτών

## Επισκόπηση

Το [Linkding](https://github.com/sissbruecker/linkding) είναι ένας ελαφρύς, μινιμαλιστικός διαχειριστής σελιδοδεικτών σχεδιασμένος για αυτο-φιλοξενία. Προσφέρει δυνατότητες όπως προσθήκη ετικετών (tagging), αναζήτηση πλήρους κειμένου, και εισαγωγή/εξαγωγή σελιδοδεικτών, επιτρέποντας την αποτελεσματική και ιδιωτική διαχείριση των σελιδοδεικτών. Ιδανικό για οικιακά εργαστήρια (home lab setups), το Linkding είναι εύκολο στην ανάπτυξη και τη συντήρηση.

## Ρύθμιση Docker Compose

Παρακάτω βρίσκεται η διαμόρφωση Docker Compose που χρησιμοποιείται για την ανάπτυξη του Linkding:

```yaml
services:
  linkding:
    image: sissbruecker/linkding:latest
    container_name: linkding
    ports:
      - "9090:9090"
    volumes:
      - ./data:/etc/linkding/data
    environment:
      LD_SUPERUSER_NAME: "username" # Αντικαταστήστε με το επιθυμητό όνομα χρήστη
      LD_SUPERUSER_PASSWORD: "securepassword" # Αντικαταστήστε με έναν ισχυρό κωδικό πρόσβασης
      LD_SUPERUSER_EMAIL: "email@example.com" # Αντικαταστήστε με το email σας
    restart: unless-stopped
    networks:
      - web

networks:
  web:
    external: true
```

## Λεπτομέρειες Διαμόρφωσης

1.  **Εικόνα (Image):**
    Χρησιμοποιεί την εικόνα Docker `sissbruecker/linkding:latest` για να εξασφαλίσει τις τελευταίες ενημερώσεις και δυνατότητες ασφαλείας.

2.  **Όνομα Container:**
    Ονομάζεται `linkding` για εύκολη αναγνώριση εντός του περιβάλλοντος Docker σας.

3.  **Θύρες (Ports):**
    Αντιστοιχίζει τη θύρα `9090` στο container με τη θύρα `9090` στον host. Πρόσβαση στο Linkding στο: `http://<host-ip>:9090`

4.  **Τόμοι (Volumes):**
    Αντιστοιχίζει το `/etc/linkding/data` στο container με το `./data` στον host, εξασφαλίζοντας επίμονη αποθήκευση για σελιδοδείκτες και ρυθμίσεις.

5.  **Μεταβλητές Περιβάλλοντος (Environment Variables):**
    Οι `LD_SUPERUSER_NAME`, `LD_SUPERUSER_PASSWORD`, και `LD_SUPERUSER_EMAIL` αυτοματοποιούν τη δημιουργία ενός λογαριασμού διαχειριστή κατά την πρώτη εκκίνηση.

6.  **Πολιτική Επανεκκίνησης (Restart Policy):**
    Το `unless-stopped` διασφαλίζει ότι η υπηρεσία επανεκκινείται αυτόματα μετά από απροσδόκητη διακοπή λειτουργίας ή επανεκκινήσεις του server.

7.  **Δίκτυα (Networks):**
    Ένα αποκλειστικό δίκτυο Docker (`web`) απομονώνει την υπηρεσία για καλύτερη ασφάλεια και ενδεχόμενη ενσωμάτωση με άλλα containers (`caddy`).

## Βήματα Ανάπτυξης

1.  **Προετοιμασία Καταλόγου Δεδομένων:**
    Δημιουργήστε τον κατάλογο για την αποθήκευση επίμονων δεδομένων:
    ```bash
    mkdir -p /docker/linkding/data
    ```

2.  **Αποθήκευση Αρχείου Compose:**
    Αποθηκεύστε την παρεχόμενη διαμόρφωση ως `docker-compose.yml` στον επιθυμητό κατάλογο.

3.  **Ανάπτυξη της Υπηρεσίας:**
    Εκκινήστε το container χρησιμοποιώντας το Docker Compose:
    ```bash
    docker compose up -d
    ```

4.  **Πρόσβαση στο Linkding:**
    Ανοίξτε ένα πρόγραμμα περιήγησης και μεταβείτε στο:
    `http://<host-ip>:9090`

5.  **Σύνδεση:**
    Χρησιμοποιήστε τα διαπιστευτήρια διαχειριστή που καθορίζονται στις μεταβλητές περιβάλλοντος για να συνδεθείτε.

## Οφέλη Αυτής της Ρύθμισης

*   **Απόρρητο:** Η αυτο-φιλοξενία εξασφαλίζει πλήρη έλεγχο των δεδομένων σας.
*   **Ευκολία χρήσης:** Απλή εγκατάσταση με αυτοματοποιημένη δημιουργία λογαριασμού διαχειριστή.
*   **Επιμονή δεδομένων:** Οι αντιστοιχισμένοι τόμοι διατηρούν τους σελιδοδείκτες ανέπαφους κατά τις ενημερώσεις ή τις επανεκκινήσεις του container.
*   **Ασφάλεια:** Το διαχωρισμένο δίκτυο Docker ενισχύει την απομόνωση και μειώνει την έκθεση σε άλλες υπηρεσίες.

## Συντήρηση

### Αντίγραφα Ασφαλείας

Δημιουργείτε τακτικά αντίγραφα ασφαλείας του καταλόγου `/docker/linkding/data` για να προστατεύσετε τους σελιδοδείκτες σας:

```bash
tar -czvf linkding-backup.tar.gz /docker/linkding/data
```

### Ενημερώσεις

Για να ενημερώσετε το Linkding, τραβήξτε την τελευταία εικόνα και επανεκκινήστε το container:

```bash
docker compose pull
docker compose up -d
```

### Κλιμάκωση (Scaling)

Ενσωματώστε το με έναν reverse proxy (π.χ. Nginx ή Traefik) για HTTPS και βελτιωμένη προσβασιμότητα.

## Συστάσεις Ασφαλείας

1.  **Χρήση ισχυρών διαπιστευτηρίων:**
    Αντικαταστήστε τα διαπιστευτήρια κράτησης θέσης με ένα ασφαλές όνομα χρήστη, κωδικό πρόσβασης και email.

2.  **Περιορισμός πρόσβασης:**
    Περιορίστε την πρόσβαση στη θύρα `9090` χρησιμοποιώντας ένα firewall εάν η υπηρεσία προορίζεται μόνο για εσωτερική χρήση.

3.  **Ασφαλείς συνδέσεις:**
    Ρυθμίστε HTTPS με έναν reverse proxy για κρυπτογράφηση της μετάδοσης δεδομένων.
